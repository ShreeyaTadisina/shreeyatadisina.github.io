Project Description

My application will make the process of procuring and processing clients for the private chef more efficient. This includes:

  • Booking an appointment
  • Contacting the chef
  • Pricing options

Target Audience: Potential customers who are interested in private chefs and meal preparation.

Overall, this is going to be a portfolio of the chef's dishes and a place for potential clients to book services.

Site Map

Wireframe & Page Designs

1. Home Page

Purpose: Introduce the private chef service, showcase the value proposition, and guide users to other pages.

Audience: Potential clients (individuals or families seeking private chef services).

Content:

  • Hook section with high-quality image of logo/food + call-to-action (e.g., "Book a Culinary Experience")
  • Brief intro (e.g., "Personalized Meals for Your Special Occasions")
  • Highlights (e.g., "Custom Menus," "Locally Sourced Ingredients")
  • Testimonials
  • Links to Services, Gallery, Contact

Interactivity:

  • Buttons: "Book your Culinary Experience" (links to Booking in Services), "Contact Chef" (links to Contact)
  • No data entry or validation needed

2. About Page

Purpose: Build trust by sharing the chef's background, philosophy, and credentials.

Audience: Potential clients evaluating the chef's expertise.

Content:

  • Chef's bio (education, experience, passion for cooking)
  • Mission statement (e.g., "Farm-to-Table with a Personal Touch")
  • Professional photo of the chef
  • Optional: Press features/awards

Interactivity:

  • No data entry
  • Button: "View My Services" (links to Services page)

3. Services Page

Purpose: Detail service offerings and pricing.

Audience: Clients deciding which service to book.

Content:

  • List of services (e.g., "Weekly Meal Prep," "Dinner Parties," "Cooking Classes")
  • Pricing (per meal/event or hourly rates)
  • Inclusions (e.g., "Grocery Shopping," "Cleanup")

Interactivity:

  • Dropdowns: "Select Service Type" (e.g., Wedding, Private Dinner)
  • Button: "Inquire Now" (links to Contact page)

4. Gallery Page

Purpose: Showcase food photography and past events.

Audience: Clients looking for visual proof of quality.

Content:

  • Photo grid/carousel of dishes/events
  • Captions (e.g., "5-Course Valentine's Dinner")

Interactivity:

  • Lightbox image zoom on click
  • Filter dropdown (e.g., "Filter by: All | Weddings | Plated Meals")

5. Contact Page

Purpose: Allow clients to send inquiries or book services.

Audience: Clients ready to engage the chef.

Content:

  • Contact form (Name, Email, Phone, Event Date, Message)
  • Chef's direct contact info (phone/email)

Interactivity:

  • Data Fields: All fields in the Contact form
  • Validations: Email format, date must be future
  • Button: "Submit" (triggers confirmation email + saves to backend)
  • Hyperlink: Link to Instagram/WhatsApp

Special Notes:

  • Navigation: Consistent header/footer across all pages
  • Mobile-Friendly: Ensure forms/gallery resize on small screens

Dynamic Functionality

JavaScript interactivity to be implemented:

Contact Form Validation

The contact page will include simple form validation to ensure users enter a properly formatted email address and required fields. When submitted, it will prevent the page from reloading and display a confirmation message. This provides instant feedback without complex coding, using basic JavaScript event listeners and validation checks.

Image Lightbox Gallery

A beginner-friendly lightbox feature will let users click thumbnail images in the gallery to view enlarged versions. The JavaScript will create a pop-up overlay with the selected image, which can be closed by clicking anywhere. This interactive element enhances the visual experience while requiring only basic DOM manipulation skills.

Manual Testimonial Slider

Testimonials on the homepage will cycle through reviews using clearly labeled "Previous" and "Next" buttons. The JavaScript stores reviews in an array and updates the display when buttons are clicked. This controlled navigation is easier for beginners than automatic sliders while still showcasing client feedback effectively.

Mobile Menu Toggle

An icon will toggle the mobile navigation visibility when clicked. The JavaScript simply switches the menu's display style between "none" and "block". This essential mobile feature requires minimal code but significantly improves the user experience on smaller devices.
